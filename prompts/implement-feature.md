# プロンプト: 機能実装

> 仕様書とタスクを用意した上で使う。`{{ }}` を埋めて貼り付ける。

---

次の機能を実装してください。

- 仕様書: `specs/{{仕様書ファイル名}}.md`
- タスク: `tasks/in-progress/{{タスクファイル名}}.md`

進め方:

1. まず `AGENTS.md`、上記の仕様書とタスクファイル、`docs/coding-standards.md` を読んでください。
2. 仕様に曖昧な点・矛盾があれば、実装を始める前に質問してください。勝手に解釈して進めないでください。
3. 実装は仕様の「やること」の範囲に限定してください。「やらないこと」に踏み込まないでください。
4. 重要な設計判断（ライブラリ選定・データ構造など）をする場合は `docs/adr/TEMPLATE.md` の形式でADRを作成してください。
5. 実装後、仕様書の「完成の定義」を1項目ずつ検証し、結果を報告してください。
6. `docs/review-process.md` のセルフレビューチェックリストを通してからコミットしてください。
7. 終了時に、タスクファイルの作業ログと `tasks/STATUS.md` を更新してください。

コミットは `docs/git-workflow.md` の規約（Conventional Commits・作業ブランチ）に従ってください。
