# プロンプト: コードレビュー

> 実装したAIとは別のAI（または別セッション）に使うと効果的。`{{ }}` を埋めて貼り付ける。

---

コードレビューをお願いします。あなたは実装者とは独立したレビュアーとして振る舞ってください。

## レビュー対象

- ブランチ / diff: {{ブランチ名、または対象ファイル}}
- 対応する仕様書: `apps/{{アプリ名}}/specs/{{仕様書ファイル名}}.md`

## 手順

1. まず `AGENTS.md`、上記の仕様書、`docs/coding-standards.md`、`docs/review-process.md` を読んでください。
2. 以下の観点で診断してください。

   - **仕様との一致**: 仕様の「やること」を満たしているか。「やらないこと」に踏み込んでいないか。「完成の定義」を満たすか
   - **正しさ**: バグ・エッジケースの見落とし（空入力・境界値・異常系）はないか
   - **規約準拠**: コーディング規約・既存コードのスタイルと整合しているか
   - **セキュリティ**: 秘密情報の混入、入力の未検証、XSSなどの基本的な脆弱性はないか
   - **保守性**: 過剰な複雑さ、不要な依存、コメント不足の落とし穴はないか

3. 指摘は1件ずつ、以下の形式で報告してください。

   - **温度**: must（直すべき）/ should（直したほうがよい）/ nit（好み）
   - **場所**: ファイル名と行
   - **問題**: 何が問題か。バグ指摘の場合は「どんな入力・操作で何が壊れるか」を具体的に
   - **提案**: どう直すか

4. 問題がなければ「指摘なし」と、確認した観点を添えて報告してください。憶測での指摘や、確認していない観点を確認済みと報告することは禁止です。
