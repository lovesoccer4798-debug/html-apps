# specs/ — 仕様書

「**何を作るか**」を記録する場所。実装の前に仕様を書く（`docs/development-flow.md` 参照）。

## 運用ルール

- 新機能・新アプリごとに `TEMPLATE.md` をコピーして1ファイル作る。
- ファイル名: `<プロジェクト名>-<機能名>.md`（例: `todo-app-core.md`）。英語ケバブケース。
- 仕様は**生きたドキュメント**。実装中に判明した変更は仕様書に反映し、実装と仕様を乖離させない。
- 「どう実装するか」は書かない（それはコードとADRの領分）。「何を・なぜ・完成の定義」を書く。
- 廃止した仕様は削除せず、冒頭に「ステータス: 廃止」と理由を書いて残す。
