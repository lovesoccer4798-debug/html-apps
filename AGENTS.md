# AGENTS.md — AI共通ルール

このファイルは、このリポジトリで作業する **すべてのAIコーディングエージェント**（Claude Code / Codex / Gemini CLI / Cursor / Windsurf / その他）に共通のルールを定める、**唯一のルールファイル**である。

各AI固有のファイル（`CLAUDE.md`、`GEMINI.md` など）はこのファイルへの参照のみを含むスタブであり、ルール本体をそこに書いてはならない。

---

## 1. 大原則：リポジトリがSingle Source of Truth

- **会話履歴に依存するな。** セッションが終われば会話は消える。仕様・判断・進捗・申し送りは、必ずリポジトリ内のファイルに書き残す。
- 「次のセッションの自分（あるいは別のAI）が、このリポジトリだけを読んで30分以内に作業を再開できるか？」を常に自問する。答えがNoなら、必要な情報をファイルに書いてから作業を終える。

## 2. 作業開始時に必ず読むもの（この順で）

1. `README.md` — 全体像
2. この `AGENTS.md` — ルール
3. `tasks/STATUS.md` — 現在の進捗スナップショット
4. 担当するタスクファイル（`tasks/in-progress/` または `tasks/backlog/`）
5. タスクが参照する `specs/` の仕様書
6. 関連する `docs/adr/` の設計判断

## 3. 情報の置き場所（ここに書け）

| 情報の種類 | 置き場所 |
|---|---|
| 何を作るか（要件・仕様） | `specs/` |
| なぜそう設計したか（判断と理由） | `docs/adr/` |
| 今なにをしているか（進捗・申し送り） | `tasks/`（特に `tasks/STATUS.md`） |
| ユーザーに見える変更の履歴 | `CHANGELOG.md` |
| 恒久的なルール・手順 | `docs/` |
| AIへの定型指示 | `prompts/` |

**迷ったら書く。** ドキュメントの過剰よりも、暗黙知の発生のほうが害が大きい。

## 4. 作業の進め方

1. **仕様を確認する。** 仕様がない機能を実装しない。仕様が曖昧なら、まず `specs/` に仕様を書く（または追記する）ところから始める。
2. **タスクを更新する。** 着手時にタスクファイルを `tasks/backlog/` から `tasks/in-progress/` へ移動し、完了時に `tasks/done/` へ移動する。
3. **小さく進める。** 1コミット＝1つの論理的変更。巨大な一括変更を避ける。
4. **重要な設計判断をしたらADRを書く。** ライブラリ選定、データ構造、アーキテクチャの選択など、あとから「なぜ？」と聞かれる判断はすべて `docs/adr/` に記録する。
5. **作業終了時に必ず `tasks/STATUS.md` を更新する。** 何が終わり、何が残り、次に何をすべきかを書く。これを怠ると次のセッションが再開できない。

## 5. Git運用（詳細は `docs/git-workflow.md`）

- コミットメッセージは Conventional Commits 形式（`feat:` / `fix:` / `docs:` / `refactor:` / `test:` / `chore:`）。
- コミットメッセージに使用したAIツール名やモデル名を含めない（ベンダー中立を保つ）。
- main へ直接コミットせず、作業ブランチを切る。
- push前にテストを実行する（`docs/testing-policy.md` 参照）。

## 6. コーディング（詳細は `docs/coding-standards.md`）

- 既存コードのスタイル・命名・イディオムに合わせる。
- コメントは「コードから読み取れない制約や意図」のみに書く。
- 秘密情報（APIキー、トークン、パスワード）を絶対にコミットしない。設定は環境変数か、gitignoreされた設定ファイルで扱う。

## 7. 禁止事項

- ❌ 特定のAIツール固有の機能（独自メモリ、独自設定形式、独自プラグイン）に依存する成果物を作ること
- ❌ 仕様・設計判断・進捗を会話の中だけで完結させ、ファイルに残さないこと
- ❌ `tasks/STATUS.md` を更新せずに作業を終えること
- ❌ ユーザーの確認なしに、破壊的変更（データ削除・履歴書き換え・公開済みAPIの破壊）を行うこと
- ❌ テストが失敗した状態を「成功した」と報告すること

## 8. ドキュメントの言語

- ドキュメント・コミットメッセージ・タスク記述は**日本語**を基本とする。
- コード内の識別子（変数名・関数名）とコードコメントは**英語**とする。

## 9. このファイル自体の変更

`AGENTS.md` の変更はワークスペース全体に影響するため、変更する場合は理由をコミットメッセージに明記し、大きな方針変更ならADRを残すこと。
