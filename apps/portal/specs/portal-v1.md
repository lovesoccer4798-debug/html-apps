# 仕様: NEST Portal v1（ホーム画面）

- **作成日**: 2026-07-07
- **最終更新**: 2026-07-07
- **ステータス**: 実装済み
- **関連**: [体験設計（Experience Architecture）の決定事項], `docs/vocabulary.md`, [ADR: Portalの配置](../../../docs/adr/20260707-portal-placement.md)

## 1. 目的（なぜ作るのか）

NESTの入口となるホーム画面。初めての人が「何がある場所か」を3分で眺められ、URLひとつ共有するだけで体験（知る→安心する）が始まる状態を作る。

## 2. スコープ

### やること

- ヘッダー: NEST文字ロゴ・Brand Philosophy・来訪者向けの一文・「はじめての方はこちら」ボタン（→START_HERE）＋GitHubへの橋の一言
- 5つの場所のカード（語彙台帳の絵文字・名称・一言説明をそのまま使用）＋「これは何？」の開閉式説明
- Appsセクション: アプリカード＋成長バッジ（🥚🐣🐦🕊）＋「次のアプリ」の空スロット
- Journey・STATUSへの導線、フッター（AT-Labo・Made in NEST・GitHubリンク）
- スマホ対応・ライト/ダーク両対応

### やらないこと（今回は対象外）

- JavaScriptによる動的機能（アプリ一覧の自動生成・検索・進捗表示）
- アクセス解析・外部サービス連携・外部CDN
- 凝ったアニメーション

## 3. ユーザーから見た振る舞い

1画面のWebページ。上から: ヘッダー（3秒で何の場所かわかる）→ 5枚カード（10秒でどこを押せばいいかわかる）→ アプリ一覧 → フッター。カードをクリックするとGitHub上の該当ドキュメントへ移動する（移動前に橋の一言で不安を予防）。「これは何？」を押すと一言解説が開く（HTML標準の`<details>`、JS不使用）。

## 4. データ・入出力

静的HTML 1ファイル（`apps/portal/index.html`）。アプリ一覧は手動更新（v1の割り切り）。リポジトリ内ドキュメントへのリンクはGitHubのblob URLを使用。

## 5. エッジケース・エラー時の挙動

- JS無効環境: 全機能が動作する（JSゼロのため）
- スマホ幅: カードが1列に折り返す
- ダークモード: `prefers-color-scheme` で自動追従

## 6. 完成の定義（Definition of Done）

- [x] URLを開いて3秒で「何の場所か」、10秒で「どこを押すか」がわかる構成になっている
- [x] 5つの場所すべてに正しいリンクと開閉式の一言解説がある
- [x] 語彙台帳と絵文字・名称・一言説明が完全一致している
- [x] スマホ幅・ダークモードで崩れない（スクリーンショットで確認）
- [x] 外部CDN・JSに依存していない
- [x] **初心者テスト**（`docs/review-process.md`）を実施し、結果を `docs/reviews/phase-3.md` に記録した
- [ ] GitHub Pagesで公開されている（人間の操作待ち: `docs/human-only-operations.md` 参照）

## 7. 未解決の論点

- アプリが10個を超えたら一覧の自動生成を検討（それまでは手動）
- 実際の初心者（人間）によるテストはシミュレーションの代替であり、機会があり次第実施する
