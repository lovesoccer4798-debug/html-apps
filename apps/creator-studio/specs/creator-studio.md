# 仕様: Creator Studio v1（SNS投稿プロンプト生成アプリ）

- **作成日**: 2026-07-08
- **最終更新**: 2026-07-08
- **ステータス**: 確定
- **関連**: `docs/development-flow.md`, `docs/design-guide.md`, `apps/portal`（先行アプリ）, `docs/philosophy/vision.md`（North Star: 公開アプリ）

## 1. 目的（なぜ作るのか）

Creator Studio は**「投稿を生成するアプリ」ではなく、「Claude・ChatGPT・Gemini などの生成AIへ渡す、高品質なプロンプトを生成するアプリ」である。** この立ち位置は、将来 API 連携版に発展しても変えない（アプリ自身は投稿を書かず、どのAIに渡しても良いプロンプトを作ることに徹する）。

エピソード・考え・音声文字起こしなどの素材を入力すると、Note / X / Threads / Instagramリール向けの投稿を作るためのプロンプトを生成する。**AI APIは使わず**、完全にブラウザ内で完結し、**無料**で使える。NESTの最初の実用アプリ（Phase 7「最初の住人」）。

## 2. スコープ

### やること
- 入力: **タイトル（テーマ）** と **エピソード本文** の2項目
- 出力媒体の選択（Phase 1は**単一選択**。Note / X / Threads / Instagramリール）
- パーソナライズ設定（口調・語尾・ターゲット・文章量・論理⇔感情・絵文字を使わない・AI感を減らす 等。データ駆動で追加可能）
- 「プロンプト生成」→ 結果表示 → ワンクリックでコピー
- **パーソナライズ設定のみ**をブラウザの LocalStorage に保存し、次回自動復元

### やらないこと（今回の対象外）
- AI APIの呼び出し（プロンプトを作るだけ。生成は各自がAIに貼る）
- ログイン・サーバ通信・クラウド保存
- **タイトル・本文の保存**（毎回まっさらから。プライバシー＝信頼境界を守る）
- 複数媒体の**同時**生成（UIは単一選択。内部構造は複数対応に備える／後述）
- 履歴・お気に入り・生成結果の保存

## 3. ユーザーから見た振る舞い

1ページ完結。上から「タイトル入力 → 本文入力 → 媒体を選ぶ → 仕上げの設定 → 生成ボタン → 結果＋コピー」。設定は前回の値が最初から入っている（LocalStorage復元）。スマホは1カラム、PCは「左で作り・右で確認」の2カラム。ダークモード自動。**絵文字は使わずLucideアイコン**で表現。

## 4. データ・入出力

- **入力**: タイトル(text)・本文(textarea)・選択媒体(1つ)・設定オブジェクト
- **出力**: 組み立てられたプロンプト文字列（タイトルを含む）
- **保存**: LocalStorage キー `creatorStudio.settings.v1` に**設定オブジェクトのみ**保存。タイトル・本文は保存しない。キーにバージョンを含め、将来の設定追加で壊れないようにする（未知キーは既定値で補完）
- **依存**: 外部CDN・フレームワーク・ネットワーク通信なし（素のHTML/CSS/JS・tokens.css継承）

### 拡張に備えた内部構造（長く育てるための設計）
- **SETTINGS[]**（設定のデータ定義）: `{id, label, type:'toggle|select|text', options, default}`。**設定追加＝配列に1行足すだけ**でUI・保存・プロンプト反映まで自動で乗る
- **MEDIA[]**（媒体のデータ定義）: `{id, label, icon, guide}`。`buildPrompt(media, state)` は**1媒体を受け取る純粋関数**にする。Phase 1は選択1件で1回呼ぶだけ。**将来の複数同時生成は「選択された媒体配列を map して buildPrompt を回す」だけで実現でき、生成ロジック本体は変更不要**

## 5. エッジケース・エラー時の挙動

- タイトルまたは本文が空 → 生成ボタン押下時に「素材を入力してください」と促す（壊れない）
- コピー未対応/不許可のブラウザ → 結果は常に `textarea` に表示し、手動選択でコピーできる
- LocalStorage が使えない/壊れている環境 → 保存はあきらめ、既定値で通常動作（機能は落ちない）
- 古いバージョンの保存データ → 未知/欠損キーは既定値で補完

## 6. 完成の定義（Definition of Done）

- [ ] タイトル・本文を入力し、4媒体それぞれでプロンプトが生成できる
- [ ] 生成プロンプトに**タイトル**が含まれる
- [ ] 設定を変えると生成プロンプトに反映される
- [ ] コピーが動く（スマホ・PC両方）
- [ ] **LocalStorageに保存した設定が、次回アクセス時に自動復元される**（タイトル・本文は復元されない）
- [ ] **生成プロンプトが特定AIに依存せず、Claude・ChatGPT・Gemini等どの主要生成AIへ貼っても自然に使える**内容である（AI固有の構文・システムトークンを使わず、汎用的な日本語の指示文にする）＝ベンダー中立
- [ ] 絵文字ゼロ・Lucideアイコン・tokens.css準拠・ダーク/レスポンシブOK
- [ ] JS/CSSに外部依存（CDN）がない

## 7. 未解決の論点・将来の追加予定

- **将来の追加予定（Phase 2以降）**: 「Output Profile」（ストーリー重視・学び重視・共感重視・営業寄り・教育寄り など、投稿の狙いのプリセット）を追加する。現行の SETTINGS 配列に項目を足すだけで拡張できる前提とする
- 媒体ごとの細部（Xの文字数上限の明記の仕方、リールの台本フォーマット）は実装しながら調整
- 「論理⇔感情」を選択式にするかスライダーにするか（初期は選択式・拡張余地は残す）
- 設定の初期プリセット（おすすめの初期値）をどこまで用意するか
