# 仕様: Task Calendar — ビジョンボード（思い出すためのページ）

- **作成日**: 2026-07-15
- **最終更新**: 2026-07-15
- **ステータス**: 確定（2026-07-15 オーナー回答反映済み・実装待ち）
- **関連**: v4仕様（ルーティン）/ v6仕様（マイカレンダー）

## 1. 目的

「人は忘れる生き物」— なぜ習慣を続けるのかを思い出し、ワクワクを保つための自由度の高いページ。写真・ことば・ゴール宣言を貼っておき、ルーティンの「なぜ」と接続する。

## 2. スコープ

### やること

- **ボード（複数作成可）**: ユーザーが自由にビジョンボードを作れる。1枚のボード＝写真・ことばを自由配置するキャンバス
- **2つのモード（作成時に選択・後から変更可）**:
  - **テンプレート型（コラージュ）**: あらかじめ用意した枠レイアウト（例: 2×2・大1＋小2・�グリッド）に写真をはめ込む。迷わず作れる
  - **自由配置型**: 写真を好きな位置・大きさ・回転でぺたぺた貼れる（付箋のように）。ドラッグで移動・つまんで拡大縮小
- **各写真の下（またはボード末尾）にテキストメモ**: そのボード／写真の「テーマ・意図・このビジョンの意味」を書ける（振り返って思い出すための言葉）
- **写真の保存はIndexedDB**（ブラウザ内の大容量保存庫・無料・オフラインOK）。保存時に長辺1280pxへ自動縮小して容量節約。目安上限30枚（超えたら古い順の整理を促す）
- **「1年前の今日」**: 振り返り画面に、1年前（なければ半年前・1ヶ月前）の「今日のひとこと」をそっと表示
- 置き場所: **ルーティンタブ内の上部切替（ルーティン｜ビジョン）** — 5タブ化を避け「なぜ→習慣」の動線が自然
- **今日画面へのランダム表示は入れない**（オーナー判断）

### やらないこと

- 写真のクラウド同期（localStorage系のエクスポートに写真は含まれない旨を明記）・コラージュ編集・手書き

## 3. データ

- `boards: [{ id, title, mode: 'template'|'free', template?, memo, order, createdAt }]`（localStorage）
- ボード内の要素: `items: [{ id, boardId, kind: 'photo'|'word', imageKey?, text?, x, y, w, h, rot }]`（自由配置は座標、テンプレート型は枠index）
- 画像本体: IndexedDB `tc-images` ストア（key=imageKey, value=縮小済みBlob）

## 4. エッジケース

- IndexedDB容量超過 → エラーを優しく表示し保存中止（アプリは壊さない）
- 画像読み込み失敗 → プレースホルダ表示＋削除導線
- ルーティン削除時 → ゴール宣言カードは「（終了したルーティン）」表示に

## 5. 完成の定義

- [ ] 3種のカードを追加・削除・並べ替えでき、リロード・オフラインでも表示される
- [ ] 写真が自動縮小されIndexedDBに保存される（localStorageを圧迫しない）
- [ ] 振り返りに「1年前の今日」が出る（データがある場合）
- [ ] 全回帰PASS・ライト／ダーク対応
