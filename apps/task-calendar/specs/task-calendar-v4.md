# 仕様: Task Calendar v4 — ルーティンパッケージ（習慣の束でゴールに向かう）

- **作成日**: 2026-07-15
- **最終更新**: 2026-07-15
- **ステータス**: 実装済み（2026-07-15）
- **関連**: v3仕様 `task-calendar-v3.md` / タスク `tasks/in-progress/20260714-task-calendar.md`

## 1. 目的（なぜ作るのか）

「マラソン大会に出る」のようなゴールに向けた習慣の束を**ルーティンパッケージ**としてひとまとめにし、有効化するだけでカレンダーに流れ込むようにする。達成の判定は完璧主義を避けた**ゆる達成**（少しでもやれたら合格・週n日でOK）で、クリアの可視化によって「自分を満たす」習慣化を支える。将来は友人とパッケージを交換できる。

## 2. スコープ

### やること

- **パッケージの作成・編集・削除**
  - タイトル（例:「ダイエット計画」）・ゴール文（例:「夏までに-3kg」）・**色**（アクセント6色から。カレンダー上での見分けに使う）
  - **週の目標日数**（1〜7日。**既定は週3日**）— 「週3日できたら今週は合格」のゆるスタート方式。毎日100%を課さない（1回の失敗で心が折れない設計）
  - **アイテム（習慣タスク）を複数**: タイトル・繰り返し（毎日/毎週/毎月）・時刻（任意）・所要時間（任意）。時間はあくまで目安 — **チェックさえ入れば合格**（=ゆる達成。「外に出ただけで成功」）
- **有効化／一時停止**: 有効にするとアイテムが繰り返しタスクとしてカレンダーに現れる（パッケージ色のタグつき）。一時停止すると**その日以降**は現れない（過去の実績は消えない）
- **ルーティン画面（ボトムナビ4つ目のタブ）**
  - パッケージ一覧カード: タイトル・ゴール・今週の進捗ドット（例: ●●●●○○○ 4/5日）・連続クリア週数
  - パッケージ詳細: アイテム一覧（編集）・達成カレンダー（そのパッケージだけのヒートマップ）・週ごとの合否履歴
- **達成の数え方（ゆる達成）**: その日、パッケージ内のアイテムを**1つでも完了したら「やった日」**。週の「やった日」が目標日数以上なら**その週は合格💮**。連続クリア週数をストリークとして表示
- **共有（エクスポート／インポート）**: パッケージをテキスト（JSON）として書き出し・読み込み。コピーして友人にLINE等で送れば、相手のアプリに同じルーティンが入る（サーバー不要・無料のまま）
- **振り返り連携**: 振り返り画面にパッケージ別の達成カードを追加
- **既存機能との関係**: パッケージのアイテムは通常の繰り返しタスクと同じ操作（チェック・スワイプ・メモ・タイマー）ができる

### やらないこと

- リアルタイムの進捗共有・フォロー機能（サーバーが必要。将来課題として明記）
- パッケージのテンプレート集の同梱（まず自分用に育ててから）
- ビジョンボード（v5で別仕様）

## 3. ユーザーから見た振る舞い

```
［ルーティン］タブ
┌──────────────────────────────┐
│ ダイエット計画        🎯 夏までに-3kg │
│ 今週 ●●●●○○○ 4/5日   連続 3週💮      │
│ ├ ☑ ウォーキング（毎日・30分・6:00）   │
│ ├ ☐ 鶏肉を食べる（毎日・夜）           │
│ └ ☐ 体重を記録（毎日）                │
│ [有効] [編集] [共有]                   │
└──────────────────────────────┘
＋ 新しいルーティン
```

- 作成: ルーティンタブの「＋」→シートでタイトル・ゴール・色・週目標→アイテムを追加していく
- 週の進捗ドットは月曜はじまり。目標日数に達した週は💮マーク
- 共有: 「共有」→JSONテキストをコピー／「読み込む」→貼り付けて自分のルーティンに追加（この時点では**無効**状態で入る）

## 4. データ・入出力

- 保存先は `nest.task-calendar.v2` に `routines` 配列を**追加**（後方互換）

```json
{
  "routines": [
    {
      "id": "r...",
      "title": "ダイエット計画",
      "goal": "夏までに-3kg",
      "color": "green",
      "targetPerWeek": 5,
      "active": true,
      "startDate": "2026-07-20",
      "pausedFrom": null,
      "createdAt": 0
    }
  ]
}
```

- アイテムは**既存のタスクにフィールド追加**で表現する: `task.routineId = "r..."`（パッケージ削除時は routineId を持つタスクも一緒に削除。完了履歴を残すか消すかは削除時に選択）
- エクスポート形式: `{ "taskCalendarRoutine": 1, "routine": {...}, "items": [...] }`（バージョン番号つき。インポート時に検証し、壊れたデータは拒否）

## 5. エッジケース・エラー時の挙動

- 一時停止→再開: `pausedFrom` 以降の発生を止め、再開で `pausedFrom` を解除（過去の doneDates はそのまま）
- インポートで同名パッケージがある場合: 別IDで追加し「(2)」を付ける（上書きしない）
- 不正なインポートテキスト: エラーメッセージを出して何も変更しない
- パッケージ削除: 確認シート（「実績も消す／実績は残す」を選択）＋「元に戻す」トースト
- アイテムが0個のパッケージ: 作成は許可（あとで足せる）。進捗は「--」表示

## 6. 完成の定義（Definition of Done）

- [x] パッケージを作成・編集・削除でき、色・ゴール・週目標が保存される
- [x] 有効化するとアイテムがカレンダーに現れ（パッケージ色タグつき）、一時停止で以降は現れない（過去実績は残る）
- [x] 「1つでも完了=やった日」「週n日で合格💮」の判定が正しく、進捗ドット・連続クリア週数が表示される
- [x] パッケージ詳細に達成ヒートマップと週ごとの合否履歴が出る
- [x] エクスポート→別ブラウザ（or シークレット）でインポートして同じルーティンが再現できる
- [x] 振り返りにパッケージ別カードが出る
- [x] v2/v3の全回帰テストがPASS、データは後方互換
- [x] ライト／ダーク両モード・スマホ幅で確認済み

## 7. 論点の決定（2026-07-15 オーナー回答）

1. ボトムナビは**4タブ**（今日／カレンダー／振り返り／ルーティン）。v5ビジョンボード時に統合を再検討
2. 「やった日」は**1つでも完了でカウント**（ゆる達成）
3. 週目標のデフォルトは**3日**（パッケージごとに1〜7日へ変更可）
4. 共有は**JSONテキストのコピー＆貼り付け**（ファイル・QRは次段階）
