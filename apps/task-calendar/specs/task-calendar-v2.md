# 仕様: Task Calendar v2 — handoffデザイン反映＋繰り返し・予定・設定

- **作成日**: 2026-07-15
- **最終更新**: 2026-07-15
- **ステータス**: 実装済み
- **関連**: v1仕様 `task-calendar-v1.md` / ADR `docs/adr/20260715-task-calendar-v2-design.md` / タスク `tasks/in-progress/20260714-task-calendar.md`

## 1. 目的（なぜ作るのか）

v1の機能（3ビュー・チェック完了・タイマー・振り返り・PWA）を、オーナーがClaude designで作成したhi-fiデザインhandoff（ブルーグレー×グリーン、モバイル基準）に載せ替え、毎日使いたくなる見た目にする。あわせて「繰り返しタスク」「タスクとは別の予定」「設定（テーマ・アクセント色）」「スワイプ操作」を加え、実運用に耐える道具にする。

## 2. スコープ

### やること

- **デザイン刷新（handoff準拠）**: 背景 `#EDF1F5`（ライト）/`#1E252D`（ダーク）、カード面・境界・文字色はhandoffのトークン表どおり。実行中カードの深緑 `#123B2A` は両モード共通。数字（時刻・タイマー・日付・統計）はモノスペース。角丸: カード16px・実行中20px・ピル・月セル12px。ボトムナビ（今日／カレンダー／振り返り）＋FAB＋ボトムシート
- **画面**: ①デイリー（時刻ラベル＋点線レールのタイムライン、実行中カード）②週間（日付ブロック＋日別完了バーの行）③月間（正方形セル・ドット・下に選択日プレビュー）④フォーカス（270pxリングの全画面タイマー）⑤振り返り（統計3カード・週の棒グラフ・できたことリスト・今日のひとことメモ）⑥**設定（新規）**
- **テーマ**: ライト／ダークの2モード。OS追従＋設定画面から手動切替（自動／ライト／ダーク）
- **アクセントカラー設定**: 月カレンダーの選択日の枠、週・日ビューのタスク選択枠、チェック・ボタン等に使うアクセント色を設定画面で6色（緑・青・ティール・紫・オレンジ・ピンク）から選べる
- **繰り返しタスク**: 追加時に「なし／毎日／毎週／毎月／毎年」を指定できる。繰り返しはルールとして保存し、各日の表示時に展開する（日ごとに個別に完了できる）。削除時は「この回のみ／繰り返しごと」を選べる。編集は繰り返し全体に反映
- **予定（イベント）**: タスクとは別の「その日の予定」を追加できる（チェックボックス・タイマーなし。タイトル＋日付＋任意の時刻のみ）
- **スワイプ操作**: タスク・予定の行を左へスライドすると「編集／削除」ボタンが現れる（PCではドラッグでも可）
- **タイマーの継続**: 実行中タスクは1つまで。実行状態を保存し、リロード・再起動後も復元する
- v1から継続: チェック完了・localStorage保存・PWA（オフライン起動）・「元に戻す」削除

### やらないこと

- Google Fonts / Material Symbols のCDN読み込み（ADR参照。端末標準フォント＋Lucideで再現）
- 予定（イベント）の繰り返し・リマインダー通知・外部カレンダー同期
- 複数タスクの同時タイマー、タスクの並べ替え
- クラウド同期・アカウント

## 3. ユーザーから見た振る舞い

- 下部ナビで「今日／カレンダー／振り返り」を移動。上部セグメントで日／週／月を切替。右上のスライダーアイコンで設定へ
- **追加**: 右下の「＋」→ボトムシート。種類（タスク／予定）・タイトル・日付・時刻（任意）・所要時間（タスクのみ任意）・繰り返し（タスクのみ）
- **実行**: タスクの▶で開始→デイリー上部に深緑の実行中カード。タップでフォーカス全画面（リング・一時停止／停止／完了）
- **編集・削除**: 行を左へスワイプ→「編集」「削除」。繰り返しタスクの削除は「この回のみ／すべて」を選ぶシートが出る
- **設定**: テーマ（自動／ライト／ダーク）とアクセント色を選ぶと即時反映・保存される

## 4. データ・入出力

- 保存先: `localStorage` キー `nest.task-calendar.v2`（v1キーがあれば初回に自動移行）
- 構造:

```json
{
  "tasks": [
    { "id": "t...", "title": "ストレッチ", "date": "2026-07-15", "time": "07:30", "minutes": 15,
      "done": false, "doneAt": null, "createdAt": 0 },
    { "id": "t...", "title": "日記", "time": "22:00", "minutes": 5, "createdAt": 0,
      "repeat": "daily", "startDate": "2026-07-15", "doneDates": {"2026-07-15": 1720000000}, "exDates": [] }
  ],
  "events": [ { "id": "e...", "title": "歯医者", "date": "2026-07-18", "time": "10:00" } ],
  "notes": { "2026-07-15": "今日のひとこと" },
  "settings": { "theme": "auto", "accent": "green" },
  "running": { "taskId": "t...", "dateKey": "2026-07-15", "totalMs": 900000, "endAt": 1720000900000, "remainingMs": null, "paused": false }
}
```

- 繰り返しの展開: daily=毎日 / weekly=開始日と同じ曜日 / monthly=同じ「日」（存在しない月はスキップ、例: 31日）/ yearly=同じ月日。`exDates` の日は出さない

## 5. エッジケース・エラー時の挙動

- v1データの移行: 初回ロード時に `nest.task-calendar.v1` を読み、v2形式に変換して保存（v1キーは残す。壊れていれば空で開始）
- 実行中タスクの復元: `endAt` を過ぎていたら「時間切れ（完了待ち）」状態で復元
- 繰り返しタスクを過去日に完了 → その日の `doneDates` にのみ記録（他の日に影響しない）
- スワイプ中の誤タップ防止: 開いた行がある間は他の行のスワイプ開始で閉じる。縦スクロールとの競合は移動方向で判定
- 空タイトル・不正な所要時間はv1と同じく拒否

## 6. 完成の定義（Definition of Done）

- [x] ライト／ダークが設定（自動／手動）で切り替わり、両モードでhandoffのトークンどおりの配色になる
- [x] 設定画面でアクセント色を変えると、月の選択日の枠・週／日のタスク選択枠・チェック等に即時反映され、リロード後も保持される
- [x] 繰り返しタスク（毎日・毎週・毎月・毎年）が各日に展開され、日ごとに独立して完了でき、「この回のみ／すべて」の削除が動く
- [x] 予定（イベント）を追加でき、タスクとは見た目が区別され、チェック・タイマーが付かない
- [x] 行の左スワイプで編集／削除が現れ、編集シートで内容を変更できる
- [x] タイマー実行中にリロードしても実行状態が復元される
- [x] v1のDoD項目（追加・完了・削除undo・保存・タイマー・PWAオフライン）がv2でも成立している
- [x] 手動確認（Chromium・ライト／ダーク／モバイル幅）が完了している

## 7. 未解決の論点

- 予定（イベント）の繰り返し対応はv3候補
- handoffの「今日のひとこと」はローカルメモのみ（振り返り画面）。書き味の改善は使いながら判断
