# ADR: ADRのファイル名を連番から日付スラッグ式に変更する

- **日付**: 2026-07-07
- **ステータス**: 採用
- **関連**: `templates/adr.md`, [ADR運用の採用](20260706-record-architecture-decisions.md)

## 背景（Context）

初期のADR採番は `0001-`, `0002-` の連番だった。共同開発（複数人・複数AIエージェントの並行作業）では、並行ブランチがそれぞれ「次の番号」を取ってしまい、マージ時に番号が衝突する。

## 検討した選択肢

### 選択肢A: `YYYYMMDD-スラッグ.md` 形式

- 長所: 並行作業で衝突しない（同日でもスラッグで区別）。時系列がファイル名ソートで保たれる
- 短所: 「ADR-7を見て」のような短い参照ができない（→ リンクで参照すればよい）

### 選択肢B: 連番を維持し「マージ時に振り直す」規則を追加

- 長所: 従来の慣習どおり
- 短所: 振り直し時に既存参照（他ドキュメントからのリンク）が壊れるリスク。手作業ルールはいつか破られる

## 決定（Decision）

選択肢Aを採用する。既存2件も新形式にリネームし、本文中の番号参照をファイルリンクに置き換えた。ADR間の参照は必ず相対リンクで行う（番号や通称での参照をしない）。

## 影響（Consequences）

- 良くなること: 採番の調整という共同開発時の摩擦が構造的に消える
- 引き受けるリスク: 特になし（採用済みADRの「編集しない」原則に対し、今回のリネームは内容不変のため例外とみなす）
- 覆す条件: なし（連番に戻す動機が想定されない）
