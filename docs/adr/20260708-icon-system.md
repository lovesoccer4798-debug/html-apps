# ADR: UIの絵文字をLucideアイコン（インラインSVG同梱）へ移行する

- **日付**: 2026-07-08
- **ステータス**: 採用
- **関連**: `docs/design-guide.md`, `docs/vocabulary.md`, [Jekyll採用](20260707-jekyll-pages.md)

## 背景（Context）

UIの絵文字はカジュアルに寄りすぎ・AI感が強く・OSごとに見え方が揃わないため、ブランドの4つの柱（シンプル・落ち着き・信頼感・長寿）に沿った統一アイコンへ移行したい（オーナー方針）。制約: 外部CDN禁止・軽量・10年運用。

## 検討した選択肢

### 選択肢A: Lucide（インラインSVG同梱）

- 長所: ISCライセンス・1,500超の線画で完全統一・プレーンSVGで`currentColor`内蔵（ダークモード自動追従）・柔らかい線がNESTの「安心」に合う。インライン同梱なら**配布元が消えても手元のSVGは残る**
- 短所: インライン分HTMLが長くなる（1個約300バイト・実測で許容）

### 選択肢B: Heroicons

- 長所: MIT・品質高い
- 短所: 約300点と少なく、将来の語彙拡張で足りなくなる可能性。Tailwind文脈の造形

### 選択肢C: Material Symbols

- 長所: 点数最大
- 短所: フォント配信前提（数百KB）か個別管理が煩雑。Googleのブランド色が強く「特定ベンダーの顔」になる

### 実装方式の比較（Lucide内）

`<img src>`参照はダークモードで色が追従しない。アイコンフォントは全部入りで重い。**インラインSVG**が依存ゼロ・色追従・自己完結で最良。

## 決定（Decision）

選択肢A（Lucide・インラインSVG同梱）を採用。あわせて**二層方針**を定める: HTML UIはアイコンへ移行、Markdown正本の絵文字はプレーンテキストとして安全なため段階削減に留める。例外として成長バッジ（🥚🐣🐦🕊）とロゴ（🪺）は意図して維持。対応表は語彙台帳、運用ルールは `docs/design-guide.md` に置く。

## 影響（Consequences）

- 良くなること: UIの見た目が端末に依存せず統一され、落ち着いた印象になる。依存ゼロは維持
- 引き受けるリスク: アイコン追加時にSVG取得の一手間（入手URLをdesign-guideに記録済み）
- 覆す条件: Lucideの保守が止まり必要なアイコンが得られなくなったとき（同梱済み分は影響なし）
