# ADR: コミットトレーラーを許容し、squashマージを基本とする

- **日付**: 2026-07-07
- **ステータス**: 採用
- **関連**: `docs/git-workflow.md`

## 背景（Context）

初期ルールは「コミットメッセージにAIツール名を含めない」だったが、ホスト型AIツール（Claude Codeのウェブ版等）は `Co-Authored-By:` などのトレーラーを**強制的に自動付与**し、除去できない。実際に本リポジトリの初回コミットがこのルールに違反する形になった。守れないルールは形骸化の起点になる。また、AI協働では細かいWIPコミットが増えやすく、mainの履歴が読みにくくなる問題もあった。

## 検討した選択肢

### 選択肢A: 「タイトル・本文には書かない、自動トレーラーは許容」に改定＋squashマージ採用

- 長所: 現実のツール挙動と矛盾しない。履歴の本文は中立を維持。squashによりmainは「1マージ=1論理変更」の綺麗な履歴になり、トレーラー付きWIPコミットもmainに残りにくい
- 短所: 個々のWIPコミットの粒度がmainから失われる（タスクファイルの作業ログで補完）

### 選択肢B: 禁止を維持し、トレーラーを手動で除去する運用

- 長所: 履歴の完全な中立性
- 短所: ホスト型ツールでは技術的に不可能な場合がある。可能な場合も毎回の手作業が必要で、確実に守られなくなる

## 決定（Decision）

選択肢Aを採用する。ルールは「実際に守れること」を優先する。中立性の実質（どのAIで作業しても本文が同じ形式）は維持される。

## 影響（Consequences）

- 良くなること: ルールと現実の乖離が解消。mainの履歴が論理変更単位になる
- 引き受けるリスク: トレーラーには特定ツール名が残る（履歴の完全な無菌化は放棄）
- 覆す条件: 全ツールがトレーラーの抑制をサポートしたら、再び全面禁止にできる
