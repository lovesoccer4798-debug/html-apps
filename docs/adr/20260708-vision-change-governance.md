# ADR: Vision専用の変更ガバナンスは作らない（思想層凍結が既に定めている）

- **日付**: 2026-07-08
- **ステータス**: 採用
- **関連**: [思想層の凍結ルール](../philosophy/README.md), [思想層の配置](20260707-philosophy-layer.md), [ルール索引](../governance.md)

> mini-ADR。

## 背景

「Future=Trigger、Phase=承認ゲート、ADR=mini-ADR と各所に変更条件があるのに、Visionだけ変更条件がない」という指摘を受け、Vision専用の変更ガバナンス（変更できる条件／してはいけないケース／手順／記録方法）を新設すべきか検討した。

## 決定

**作らない。** Visionは思想層6文書の1つであり、思想層の凍結ルール（`philosophy/README`）が既に4点すべてを定めている: 変更できる条件＝Version 2.0の節目／してはいけない＝Version 1の間（凍結）／手順＝憲法改正／記録＝ADR必須。専用ガバナンスを足すと凍結ルールと二重管理になり、片方が古びてSSOTが壊れる。

「Visionだけ無governance」に見えた真因は、その governance が philosophy/README に埋もれて発見しにくかったこと。これはルール索引（`docs/governance.md`）が思想層の変更ルールの在り処を指すことで解消。あわせて凍結ルール側に「Visionを含む6文書に適用」と明記した（新ルールではなく、既存ルールの明確化）。

## 影響

- 良くなること: 変更条件の二重管理を避けられる。Visionの変更ルールが索引から辿れる
- リスク: なし（既存ルールの範囲内）
- 覆す条件: 将来 Version 2.0 の憲法改正プロセスを詳細化する必要が出たら、そのとき philosophy 側で定義する（Vision単独ではなく思想層全体として）
