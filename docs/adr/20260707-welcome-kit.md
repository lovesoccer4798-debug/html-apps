# ADR: 歓迎キット（Phase 1）の設計判断 — 自己完結プロンプト・はじめてモード・語彙台帳

- **日付**: 2026-07-07
- **ステータス**: 採用
- **関連**: `START_HERE.md`, `prompts/hello-nest.md`, `prompts/first-app.md`, `docs/vocabulary.md`, [思想層の配置](20260707-philosophy-layer.md)

## 背景（Context）

体験シミュレーション（NEST Experience Review 1.0）で、初見ユーザーの動線に3つの深刻な問題が見つかった: (1)新規者向けの挨拶プロンプトが無くresume-workが誤動作する (2)Portal/READMEからGitHubのUIに放り出される「崖」がある (3)AIを持っていない人の導入手順が無い。また正式開発フロー（仕様書→タスク→実装）が最初のアプリには重すぎ、「最初のできた！」を遅らせていた。

## 検討した選択肢と決定

### 決定1: hello-nest.md は自己完結型にする

リポジトリを読めないチャットAIに貼っても憲法どおり振る舞えるよう、NESTの説明と六箇条の要約をプロンプト内に埋め込む。対案の「AGENTS.mdを読ませる方式」は、ファイルを読めるAIでしか動かず、初心者が最初に使うのはチャットAIである可能性が高いため却下。重複（憲法の要約がプロンプト内に存在）は、思想層凍結により乖離リスクが低いと判断して許容。

### 決定2: 「はじめてモード」を法律（development-flow.md）に正式条文化する

Journey Level 3の最初の1個に限り仕様書・タスクを免除。Journey（思想層・地図）には案内のみを書き、免除規定の法源はHandbook側に置く（思想と規則の分離）。対案の「例外を作らず全フロー適用」は、最初の成功体験を書類仕事の後ろに置くことになり、Design Principles原則2に反するため却下。

### 決定3: 共通語彙台帳（docs/vocabulary.md）を新設する

5つの場所・同伴者・成長バッジの名前/絵文字/一言説明を1ファイルに固定し、全ドキュメントはここから引用する。対案の「Brand Book付録に書く」は、思想層がVersion 1系で凍結されたため、育てる必要のある語彙（バッジ追加等）を凍結層に置けず却下。

### 決定4: GitHubへの「橋」は独立ページにせず、START_HERE冒頭に置く

初見者は既にGitHub上でSTART_HEREを開いているため、最初の30秒で「この画面は怖くない」と伝えるのが最速。ページを増やさない（原則7）。

## 影響（Consequences）

- 良くなること: 初見の人が15分で「AIと挨拶→Journey Level 1」に到達できる動線が完成する
- 引き受けるリスク: hello-nest内の憲法要約は思想層改訂（Version 2.0）時に手動同期が必要（凍結ルールにより頻度は極小）
- 覆す条件: 全主要AIがリポジトリ読み込みを標準対応したら、自己完結型をやめて参照型に戻せる
