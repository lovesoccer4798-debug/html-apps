# ADR: 仕事の分類が非自明なら、AIは提案して止まる

- **日付**: 2026-07-08
- **ステータス**: 採用
- **関連**: `AGENTS.md`（8. 禁止事項）, [Phase変更ガバナンス](20260708-roadmap-change-governance.md)

> mini-ADR。

## 背景

Phase変更ガバナンス（前ADR）は「Phase構成の変更」を承認ゲートにしたが、「そもそもこの仕事はPhaseか？ ADRだけか？ 運用変更か？」という**分類そのもの**をAIが単独で決める抜け穴が残った。実際に前回、運用ルールの追加を「これはPhaseではない」とAIが自己判断し、承認前に実装・マージまで進めた。承認前に実装が済む運びでは、分類ミスが既成事実化する。

## 決定

**分類が非自明な新規の仕事は、AIが分類案を添えて提案し、承認まで実装しない。** 麻痺を防ぐため自律ゾーンを明示する。

- **提案して止まる**: 新しい能力・機能領域になりうるもの、ワークスペース全体に効く構造/ルール変更、分類が曖昧なもの
- **自律でよい**: 誤字・リンク修正、STATUS/CHANGELOG更新、承認済みPhaseの実装、単一のmini-ADRで足る小決定、質問への回答
- **決め手**: 迷ったら提案する（曖昧なケースは承認側へ倒す）

## 影響

- 良くなること: 仕事のスコープ判断がオーナー主導になり、Phase承認ゲートの抜け穴が塞がる
- 引き受けるコスト: 非自明な新規作業で承認の1往復。ただし自律ゾーンを明示したので日常作業は止まらない
- 覆す条件: 承認往復が過剰で開発を妨げたら、オーナーが自律ゾーンを広げる
