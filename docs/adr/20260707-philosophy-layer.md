# ADR: 思想層6文書をdocs/philosophy/に一次文書として配置する

- **日付**: 2026-07-07
- **ステータス**: 採用
- **関連**: `docs/philosophy/`, [AGENTS.md一元化](20260706-ai-agnostic-workspace.md)

## 背景（Context）

NESTブランドの立ち上げに伴い、判断のよりどころとなる思想文書6点（Story・Manifesto・Vision・Brand Book・Design Principles・Journey）が完成した。これらは今後すべての成果物の最上位判断基準（憲法と定規）となるため、置き場所・構成・他文書との関係を確定する必要があった。

## 検討した選択肢

### 選択肢A: `docs/philosophy/` に6文書＋玄関READMEを置く

- 長所: 「思想＝philosophy」という普遍的な名前で10年耐久。docs/配下なのでHandbookから自然に参照できる。玄関READMEが役割地図と改訂ルールを一元管理できる
- 短所: docs/直下のファイル群より一段深くなる

### 選択肢B: リポジトリ直下に `BRAND.md` 等として置く

- 長所: 目立つ
- 短所: ルートが6ファイル増えて散らかる（50プロジェクト構想と逆行）。READMEとの役割が衝突する

### 選択肢C: `docs/brand/` に置く

- 長所: 短い名前
- 短所: Design PrinciplesとJourneyは「ブランド」より広い概念であり、名前が中身を包含しない

## 決定（Decision）

選択肢Aを採用する。あわせて以下を定める。

1. **思想層は6文書で完結**とし、7つ目は作らない（問い: なぜ生まれた／何を信じる／どこへ向かう／人としてどうある／物としてどうある／人はどう育つ、で完備）
2. **Journeyは「地図の定義」のみを思想層に置く**。個人の進捗はファイル化せず `tasks/STATUS.md` に一行書く（思想層＝不変、進捗＝毎日変わる、を混ぜない）
3. **AGENTS.md（法律）に憲法参照を追加**し、AIも価値判断はBrand Book・Design Principlesに従うことを明文化する
4. 全文書に「ここは何の場所か（冒頭）」「次はどこへ（末尾）」のナビを付ける（Design Principles原則5の自己適用）

## 影響（Consequences）

- 良くなること: すべての制作判断に参照可能な基準ができ、AI・人間・セッションを問わず判断がぶれなくなる
- 引き受けるリスク: 思想文書のメンテナンスコスト（改訂ルールを玄関READMEに明記して抑制）
- 覆す条件: 思想層の文書が読まれず形骸化した場合、より短い形式への統合を検討する
