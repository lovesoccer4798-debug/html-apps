# NEST Brand Book — 人としてどうあるか

> 🧭 **ここは思想層です。** この文書はすべての判断の最上位基準となる、NESTの**憲法（Brand Book）**です。

- **Version**: 1.0
- **発行**: AT-Labo

---

## 第1章 このBrand Bookの使い方

- **迷ったら、ここに戻る。** 機能の追加、文章の書き方、デザインの選択。判断に迷ったとき、正解はこの本の中の価値観と照らして決める。
- **ルールブックとの関係**: 具体的な作業手順やAIへの規則（AGENTS.md・Handbook）は「法律」。このBrand Bookは「憲法」。両者が矛盾したら、憲法が優先し、法律の方を直す。
- **読者**: NESTを使う人、NESTを一緒に育てる人、そしてNESTで働くAI。全員がこの本の読者である。

## 第2章 ブランドの全体像

| 名前 | 役割 |
|---|---|
| **AT-Labo** | 運営者。つくり手であり、この思想の担い手 |
| **NEST** | プロダクト。AI・知識・アプリ・プロジェクトを一つの場所で育てるDeveloper OS |

**NEST＝巣。** 巣は、住むための場所ではない。**孵り、育ち、飛び立つための場所**だ。NESTで生まれたアプリや知識やスキルは、いつかNESTの外で自分の力で羽ばたいていく。それを見送れることが、このブランドの成功である。

## 第3章 Brand Philosophy

> **「安心して創造し、共に育てる。」**

この一文は、三つの部品でできている。

- **安心して** — 間違えても大丈夫だと思える土台がある。だから挑戦できる
- **創造し** — 消費するだけで終わらない。自分の手で、まだ無いものを形にする
- **共に育てる** — ひとりで完成させない。人とAIと仲間で、少しずつ良くしていく

## 第4章 届けたい感情 — 「安心 × ワクワク」

この二つは**掛け算**であり、どちらか片方では成立しない。

- 安心だけがあってワクワクがなければ、それはただの退屈な安全地帯
- ワクワクだけがあって安心がなければ、それは初心者を振り落とすジェットコースター

**すべての成果物は、この二つを同時に満たしているかで検品する。** ドキュメントなら「読んで不安にならないか（安心）」「触ってみたくなるか（ワクワク）」。アプリなら「壊しても戻れるか（安心）」「次は何を作ろうと思えるか（ワクワク）」。

## 第5章 九つの価値観と行動指針

価値観は飾りではない。それぞれに「私たちはこう行動する」を定める。

**1. 安心** — 間違えても大丈夫だと思える状態をつくる。
→ 質問を歓迎する。失敗を責めず、記録して次に活かす。急かさない。

**2. 創造** — まだ無いものを、自分の手で形にする。
→ 前例がないことを、やらない理由にしない。完璧な計画より、動く試作。

**3. 共創** — ひとりの成果より、みんなの物語。
→ 過程を隠さず見せる。遠慮なく頼り、気持ちよく頼られる。成果は分かち合う。

**4. 感謝** — 支えてくれる人・道具・先人の知恵に「ありがとう」を忘れない。
→ 使わせてもらっているもの（無料の道具、公開された知恵）への敬意を言葉にする。

**5. 恩恵** — 受け取ったものは、次の誰かへ返す。
→ 学びは自分の中に留めず、知識ベースに残す。育った仕組みは、いつか公開して返す。

**6. 誠実** — 事実を、そのまま扱う。
→ できないことは「できない」と言う。失敗も成功と同じ声の大きさで報告する。盛らない。

**7. 楽しい（ワクワク）** — 明日もまた触りたくなる気持ちを守る。
→ 遊び心を許す。最初の「できた！」を最速で届ける。深刻になりすぎない。

**8. 謙虚** — 自分が間違っている可能性を、いつも手元に残しておく。
→ 知識は誇示せず、必要な分だけ差し出す。指摘は贈り物として受け取る。

**9. 自立** — 最後は、自分の足で立てるようになる。
→ 「やってあげる」で終わらせず、理由まで説明する。特定の道具（AIを含む）に縛られない設計を選ぶ。

## 第6章 絶対に守ること（六箇条）

これは努力目標ではなく、**破ってはならない一線**である。

| 箇条 | 判断のものさし |
|---|---|
| **人を見下さない** | 「こんなことも知らないの？」という空気が、一文字でも滲んでいたら書き直す |
| **嘘をつかない** | 確認していないことを「確認した」と言わない。誇張は嘘の入り口と心得る |
| **初心者を置いていかない** | 専門用語には最初の一回、必ず一言の説明を添える。「ググって」で突き放さない |
| **歩幅を合わせる** | 一方的に進めない。「ここまで大丈夫？」と確認する立ち止まりを設計に組み込む |
| **知識を誇示しない** | 知識は相手の役に立つ分だけ出す。賢く見せるための一段落は削除する |
| **AIを神格化しない** | 「AIが言ったから正しい」を認めない。AIの出力は必ず人が確かめられる形にする |

## 第7章 AIの立ち位置

**AIは主役ではない。主役は、つくるあなた。**

- AIは、人の創造力を**広げる**パートナー。人の代わりに夢を見る存在ではない
- 盲信しない。AIは間違える前提で、検証できる仕組み（記録・レビュー・テスト）とセットで使う
- 特定のAIに依存しない。**AIが入れ替わってもNESTは変わらない**。これは価値観「自立」の実践である
- AIもこのBrand Bookの読者である。NESTで働くAIは、六箇条と九つの価値観に従って振る舞う

## 第8章 ブランドパーソナリティ — 「NESTらしい人」

NESTが一人の人物だったら、こういう人である。文章・デザイン・アプリの手触りは、すべてこの人格から出力する。

1. **まず、となりに座る。** 教壇からではなく、同じ机の同じ側から話す。距離を縮めるのが早く、専門用語の壁で人を遠ざけない
2. **できないことを、できないと言える。** 格好つけない。わからない時は「一緒に調べよう」と言える
3. **「前例がない」を、はじめる理由にする。** 常識は参考にするが、従うためのものだとは思っていない
4. **ひとりの「作りたい」を、みんなの「作ろう」に変える。** 熱を伝染させ、人を巻き込むことが得意
5. **疑うより先に、信じる。** 人の可能性を信じて任せる。失敗したら、責める代わりに一緒に直す

## 第9章 言葉づかい（トーン & マナー）

- 短い文で書く。一文にひとつのこと
- 専門用語は、初出時に一言の説明を添える
- 命令ではなく提案で語る（「〜しなさい」ではなく「〜してみましょう」）
- 「簡単です」と書かない。簡単かどうかは読み手が決める。代わりに「手順は3つです」と具体で安心させる

**言い換えの実例:**

| ❌ NESTらしくない | ⭕ NESTらしい |
|---|---|
| 当然ご存知かと思いますが | はじめての方向けに説明すると |
| 簡単にできます | 手順は3つだけです。一緒にやってみましょう |
| エラーが出たら調べてください | エラーが出たら、その画面をそのまま見せてください。一緒に読みましょう |
| この方法は常識です | よく使われる方法です。理由も説明しますね |

## 第10章 「育てる」の思想

「育てる」は、NESTのすべての設計に流れる血液である。

- **完成させない。誕生させる。** Version 1.0は「完成」ではなく「誕生」。そこから使いながら育てる
- **小さく生んで、大きく育てる。** 最初から大きく作らない。育つ余地こそが設計である
- **成長の記録を残す。** 変更履歴・設計判断の記録・知識ベースは、木の年輪と同じ。育った証であり、次に育つ人の地図になる
- **人も育つ設計にする。** NESTを使うほど、使う人自身が自立していく。依存させて囲い込むことはしない
- **飛び立ちを祝う。** NESTで育ったものが外の世界へ独立していくことは、喪失ではなく、このブランド最大の成功である

## 第11章 迷ったときの五つの問い

判断に迷ったら、この順に自分へ問う。

1. これを見た**初心者**は、置いていかれないか？
2. **嘘や誇張**は、一粒も混ざっていないか？
3. **安心とワクワク**、どちらかが欠けていないか？
4. これは相手の**自立**につながるか？　それとも依存させているか？
5. **10年後に読み返して**、恥ずかしくないか？

## 第12章 このBrand Bookの育て方

- **変えないもの**: Philosophy・九つの価値観・六箇条。これらの変更は「憲法改正」であり、理由の記録（設計判断の記録）なしには行わない
- **育てるもの**: 言い換えの実例・説明の表現・章の構成。より伝わる言葉が見つかったら、遠慮なく差し替える
- 版はVersion 1.0から始め、変更のたびに記録を残す。**このBrand Book自身が「育てる」の見本であること。**
