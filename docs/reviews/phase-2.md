# Phase 2 レビュー記録 — 安心の備蓄

- **日付**: 2026-07-07
- **成果物**: `knowledge/`（README＋種まき5件）、`docs/README.md`（Handbook表紙）、`docs/glossary.md`、`docs/human-only-operations.md`、`docs/starter-kit.md`、`docs/reviews/`（本制度）、[ADR: レビュー記録とKnowledge](../adr/20260707-reviews-and-knowledge.md)

## 目的

初めて困った人が5分以内に「先輩の転んだ跡」を見つけられる状態を作る（Experience Review P6・P8への対応）。あわせてオーナー提案3件（レビュー文化・Vision余白・Starter Kit=DNA）を処理する。

## レビュー結果

- Knowledge未開設・実体験の知見がセッション履歴に散在・用語集なし・Handbook表紙なし・人間専用操作が暗黙知、の5点を確認
- オーナー提案3件を審査（下記）

## 採用

- **提案①（レビュー文化）**: `docs/reviews/` を新設し、Phase 0・1を遡って記録。書式は5項目（目的・レビュー結果・採用・却下・理由）に固定
- **提案③（Starter Kit=DNA）**: `docs/starter-kit.md` で定義。「複製されるのは思想と仕組み、複製されないのは記憶」— テンプレートではなく、NESTを再現する最小単位として定義
- Knowledge書式: 冒頭に「状況→結論→理由」の3行サマリを義務化（5分で見つけ30秒で判断できるため）
- 種まきは実際に起きた出来事に限定（創作しない — 価値観「誠実」）

## 却下

- **提案②（VisionのCreator OS余白）は「変更不要」と判定**: Vision本文は「つくること・創造・自分の道具」で書かれており、Developer・アプリに限定する語が本文にない。唯一の限定語はNorth Star指標名「公開されたアプリの数」だが、定義列は既に「創造が世に出た数」で作品一般を包含している。Creator OSへ育った時点で「アプリ→作品」の一語改正（Version 2.0）で足りる。凍結直後の再改正はルールの重みを損なうため行わない
- knowledgeのカテゴリ分け・タグ付け（件数が少ないうちは1階層＋Grepで十分。原則7）

## 理由

歓迎キットで人が入れるようになった直後に必要なのは「最初に困った瞬間の受け皿」。Phase 3（Portal公開）は「空の状態で公開しない」（体験設計の不変条件5）ため、備蓄が公開の前提条件になる。
