# Phase 3 レビュー記録 — Portal

- **日付**: 2026-07-07
- **成果物**: `apps/portal/`（index.html・README・CHANGELOG・仕様書）、ルートのリダイレクト`index.html`、初心者テストの正式文化化（`docs/review-process.md` §3）、[ADR: Portalの配置](../adr/20260707-portal-placement.md)

## 目的

NEST初のアプリとしてPortalを**正式フロー（仕様書→タスク→実装→レビュー）で**開発し、URLひとつで初見者の体験が始まる状態を作る。あわせてオーナー提案（DNAへのレビュー文化追加・記録3兄弟の3行説明維持・初心者テストの文化化）を反映する。

## レビュー結果

**初心者テスト（シミュレーション）**: 本物の初心者が未確保のため、規定4項に基づき**初見視点のシミュレーションで代替**（スクリーンショット3種: デスクトップ・スマホ幅375px・ダークモード）。

- ✅ 3秒で「何の場所か」（ロゴ＋Philosophy＋来訪者向けの一文が最初の画面に収まる）、10秒で「どこを押すか」（緑のボタンが唯一の強調）
- ✅ 語彙台帳との絵文字・名称・一言説明の完全一致を目視確認
- 🔧 **発見→即修正2件**: ①ダークモードで説明文のコントラストがやや低い → 文字色を明るく調整 ②スマホ幅でボタン文字が窮屈 → 420px以下でサイズ調整。修正後に再スクリーンショットで確認済み
- ⏳ **実際の人間による初心者テストは未実施**。オーナーが対象者を確保でき次第実施し、結果をここに追記する

## 採用

- Portalの置き場所は `apps/portal/`（コロケーション維持）＋ルートに5行のリダイレクトindex.html
- **JavaScriptゼロ**実装（開閉はHTML標準の`<details>`。JS無効環境でも全機能動作）
- 初心者テストを `docs/review-process.md` に正式条文化（シミュレーション代替時は明記義務）
- Starter Kit DNAに「レビュー文化」（書式README＋初心者テスト規定）を追加。記録の中身は記憶側
- 記録3兄弟の3行説明（ADR＝なぜそう決めたか／Reviews＝あとから振り返る記録／Knowledge＝困った時の助け）を用語集とHandbook表紙に固定

## 却下

- Portalをリポジトリ直下に置く案（アプリ同梱構造の例外を作らない。リダイレクトで短いURLは確保できる）
- アプリ一覧の自動生成（アプリ10個までは手動。原則7）

## 理由

Portalを「NESTの正式な作り方で作られた最初のアプリ」にすることで、開発フローそのものの実証（ドッグフーディング）になる。仕様書の完成の定義のうち「GitHub Pagesで公開」のみ人間専用操作のため未完了 — 有効化手順は `docs/human-only-operations.md` に記録済み。
