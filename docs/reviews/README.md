# docs/reviews/ — フェーズレビューの記録

> 🧭 ここは**設計思想が育った歴史**の置き場です。Phaseごとのレビュー結果と、何を採用し・何を却下し・なぜそうしたかを記録します。

## ADRとの違い

| | ADR（`docs/adr/`） | レビュー記録（ここ） |
|---|---|---|
| 記録するもの | 個別の技術・設計判断 | Phase単位のレビューと意思決定の全体像 |
| 粒度 | 1判断1ファイル | 1Phase1ファイル |
| 問い | なぜこの設計にしたか | この期間に何を検討し、どう育ったか |

## 書式（これ以上増やさない）

各記録は次の5項目だけで書く: **目的／レビュー結果／採用／却下／理由**。
ファイル名は `phase-N.md`。レビューにはオーナー・AI・外部AI（ChatGPT等）の所見を含めてよい。
