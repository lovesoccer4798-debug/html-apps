# 用語集（glossary）

> 🧭 ここはNESTで出会う言葉の**やさしい辞書**です。五十音・ABC順ではなく「出会う順」に並んでいます。覚えなくて大丈夫、出会ったら引きに来てください。

## GitHubまわり

- **GitHub（ギットハブ）** — 世界中の人が作ったものを保管・公開できる、巨大な倉庫サイト。NESTの本体はここに置いてある
- **リポジトリ** — 倉庫の中の、ひと部屋。このNESTも1つのリポジトリ
- **Git（ギット）** — 変更をぜんぶ記録してくれる仕組み。これがあるから、いつでも前の状態に戻れる（＝壊しても大丈夫の正体）
- **コミット** — 「ここまでの変更を記録します」という保存の単位。ゲームのセーブポイントのようなもの
- **push（プッシュ）** — 手元の記録をGitHub（倉庫）へ送ること
- **ブランチ** — 作業用の分身。本体（main）を汚さずに試して、良ければ合流させる
- **main（メイン）** — ブランチの本体。いつも動く状態に保つ約束の場所
- **PR（プルリクエスト）** — 「この変更を本体に入れていい？」という提案。レビューしてから合流できる
- **マージ** — 提案された変更を本体に合流させること
- **squash（スカッシュ）マージ** — 細かいセーブをひとまとめにしてから合流させる方式。本体の履歴がきれいに保たれる
- **タグ／リリース** — 「この時点がバージョン1.0.0」という記念の目印
- **CI（シーアイ）** — pushのたびに自動で走る検査係。NESTでは「秘密情報が混ざってないか」「リンクが切れてないか」を見張っている
- **GitHub Pages（ペイジズ）** — リポジトリの中身をWebページとして無料公開できる仕組み

## NESTまわり

- **Markdown（マークダウン）** — `#` や `-` などの簡単な記号で書く文章形式。NESTの文書はぜんぶこれ。メモ帳でも開ける
- **プロンプト** — AIへの声かけ文。`prompts/` にコピペで使えるものが揃っている
- **ADR** — 設計判断の記録。「なぜこうしたか」を未来の人（未来の自分）のために残すメモ
- **SSOT（エスエスオーティー）** — Single Source of Truth。「正しい情報の置き場所は1つだけにする」という約束。NESTではリポジトリがそれ
- **思想層／体験層／実体層** — NESTの3階建て構造。考え方（変わらない）／案内（迎える）／中身（育つ）
