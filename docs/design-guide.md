# デザインガイド — NESTの見た目の育て方

> 🧭 ここはNESTの**ビジュアル（見た目）の方針書**です。Portal・Dashboard・今後のすべてのアプリのUIは、このガイドに従います。思想は[Brand Book](philosophy/brand-book.md)（凍結）、見た目はこのガイド（育てる）と役割を分けています。

## 4つの柱

**シンプル・落ち着き・信頼感・長く使えること。** 派手さより、10年触っていて疲れないこと。

## アイコン — Lucide を採用

絵文字はカジュアルに寄りすぎ、OS・ブラウザごとに見え方も揃わないため、UIでは線画アイコンへ移行する。採用は **[Lucide](https://lucide.dev)**（ISCライセンス）。

| 比較 | Lucide | Heroicons | Material Symbols |
|---|---|---|---|
| ライセンス | ISC | MIT | Apache 2.0 |
| 点数・一貫性 | 1,500超・線画で完全統一 | 約300 | 膨大（フォント配信前提） |
| 同梱しやすさ | ◎ プレーンSVG・`currentColor`内蔵 | ○ | △ |
| ブランド適合 | ◎ 柔らかい線画＝安心・落ち着き | ○ | △ Google色が強い |

### 使い方のルール（10年運用の生命線）

1. **CDNを使わない。** SVGをHTMLに**インラインで同梱**する（Lucideのサービスが消えても、手元のSVGは永遠に残る）
2. `stroke="currentColor"` のまま使う（ダークモードに自動追従）
3. 入手先: `https://raw.githubusercontent.com/lucide-icons/lucide/main/icons/<名前>.svg`
4. どの場所にどのアイコンを使うかは [共通語彙台帳](vocabulary.md) の対応列が正本

## 絵文字の扱い（二層方針）

| 層 | 方針 |
|---|---|
| **HTML UI**（Portal・Dashboard・アプリ） | アイコンへ移行済み。新規UIも絵文字を使わない |
| **Markdown文書**（正本） | プレーンテキストとして10年安全なため急いで排除しない。段階的に減らし、場所の識別子など最小限に留める |

### 例外（意図して残す絵文字）

- **成長バッジ 🥚🐣🐦🕊** — 「育てる」世界観の中核表現。アイコンでは温度が出ない
- **ロゴ 🪺** — 暫定のブランドマーク。正式ロゴを作る日まで維持

## Design Tokens（デザインの土台）

見た目の「値」は **`tokens.css`** に集約する。これがNESTの配色・形の**唯一の正本**であり、`style.css`（コンポーネントの見た目）はこの値を消費するだけにする。役割分担は「思想＝Brand Book（凍結）／方針＝このガイド／値＝tokens.css」。

### トークン一覧

| トークン | 意味 |
|---|---|
| `--nest-bg` | 基本の背景（Portal＝迎える面。クリーム） |
| `--nest-bg-alt` | 作業面（Dashboard＝毎日のホーム。わずかに沈めた色） |
| `--nest-ink` / `--nest-ink-soft` | 本文／補助テキスト（濃茶） |
| `--nest-card` / `--nest-line` | カード面／罫線 |
| `--nest-accent` / `--nest-accent-ink` | アクセント（巣の緑）／その上の文字 |
| `--nest-amber` | 補助アクセント（琥珀） |
| `--nest-shadow` | 影（弱く。落ち着き） |
| `--nest-radius` / `--nest-radius-pill` | 角丸／ピル形 |

### 使い方のルール

1. **色・角丸を直値で書かない。** 必ず `var(--nest-*)` を使う（1箇所直せば全体が揃う）
2. HTMLは `tokens.css` を `style.css` より**先に**読み込む
3. ライト／ダークは tokens.css 側で完結（コンポーネント側にダーク分岐を書かない）
4. **新しいアプリは `tokens.css` をコピーして始める**（Starter KitのDNA。`templates/app/tokens.css` が雛形）。文字は端末標準の日本語フォント（Webフォントを読み込まない — 軽量・依存ゼロ）

### 今はまだ作らないもの（原則7）

余白スケール・文字スケールのトークン化は、コンポーネントを整理する必要が出た時に追加する。使わないトークンを先回りで置かない。

## Portal と Dashboard の役割を、デザインで伝える

| | Portal | Dashboard |
|---|---|---|
| 役割 | 初めて来た人の入口（迎える） | 全部への近道（作業場のリンク集。毎日のホームはHandbook — [語彙台帳](vocabulary.md)の4つの役割参照） |
| 背景 | `--nest-bg`（明るいクリーム） | `--nest-bg-alt`（わずかに沈めた作業面） |
| 密度 | ゆったり・大きなヒーロー・「これは何？」の学びつき | ぎゅっと・リンク一覧・説明は最小 |

同じトークン上での差なので、ブランドは割れない。色と密度だけで「入口」と「作業場」が自然に伝わる。

## Dashboard を肥大させない（ガバナンス）

Dashboardは**入口に徹し、増やさない**。

- グループは6つで固定（いまを知る／つくる／AIと／しらべる／まなぶ／迷ったら）
- 1グループのリンクは**最大5本**まで
- 新しい情報は Handbook か Knowledge に置き、Dashboardからは既存グループのリンクで指す。**Dashboardにグループやセクションを足さない**
- 5本を超えそうになったら、それは「情報の置き場所を見直す合図」（Dashboardの問題ではない）

## このガイドの育て方

見た目の変更は「一度に全部」ではなく、触ったページから順に揃えていく（ブランドを育てるリファクタリング）。大きな方針転換はADRを残す。
