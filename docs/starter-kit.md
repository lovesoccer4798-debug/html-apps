# Starter Kit — NESTのDNA

> 🧭 ここはStarter Kitの**定義書**です。実装はまだ先（Phase 4以降）ですが、「何であるか」をここで固定します。

## 定義

Starter Kitは、テンプレートではない。**NESTを再現するための最小単位 — NESTのDNA**である。

テンプレートは「形」を複製する。DNAは「育ち方」を複製する。Starter Kitから生まれた新しい巣は、AT-Laboの巣と同じ形にはならないが、同じ思想で育ち、同じように人を迎え、いつか同じように次の巣を生む。

## 複製されるもの／されないもの

| | 内容 | 例え |
|---|---|---|
| **複製される（DNA）** | 思想層6文書・Journey・START_HERE・AGENTS.md＋スタブ・templates・prompts・CI・語彙台帳・空のapps/とknowledge/・**レビュー文化**（書式を定めた `docs/reviews/README.md` と初心者テストの規定） | 育ち方の設計図 |
| **複製されない（記憶）** | AT-Laboのapps・knowledgeの中身・STATUSの中身・レビュー記録の中身（phase-N.md）・CHANGELOG履歴・Story※ | その巣が生きた人生 |

※ Storyは特別扱い: AT-LaboのStoryは複製せず、代わりに「あなたのStoryを書くための空白ページ」を同梱する。**新しい巣には、新しい生まれた理由がある。**

## この定義から自動的に決まること

1. **汎用と固有を混ぜない**という現行の設計原則（アプリ同梱構造・思想層の分離）は、そのままDNA抽出の準備である。日々の開発でこの分離を守ることが、Starter Kitの実装コストを下げ続ける
2. Starter Kitの成功指標は配布数ではなく、**そこから生まれた巣の数**（Vision「North Star」の第3指標）
3. 形式はGitHubのTemplate Repository（無料・Use this templateボタン）を予定。ただしDNAの定義はこのファイルであり、配布手段が変わっても定義は変わらない

## まだ決めないこと（Phase 4で決める）

同梱ドキュメントの文言調整（AT-Labo固有表現の中立化）、セットアップ手順、リポジトリ名（`nest-starter` 仮）。
