# tasks/ — タスク管理

「**今なにをしているか**」を管理する場所。GitHubのIssueや外部タスクツールに依存せず、リポジトリ内で完結させる（SSOT原則）。

## 構成

```
tasks/
├── STATUS.md      # 現在地スナップショット＝プロジェクト索引（毎セッション更新・最重要）
├── backlog/       # 未着手のタスク
├── in-progress/   # 進行中のタスク（1人・1エージェントにつき1つまで）
└── done/          # 完了したタスク
```

## 運用ルール

1. タスクは `templates/task.md` をコピーして `backlog/` に作る。ファイル名: `YYYYMMDD-<アプリ名>-<短い英語スラッグ>.md`（例: `20260707-todo-app-mvp.md`。横断タスクはアプリ名の代わりに `workspace`）。
2. 着手したら `in-progress/` へ、完了したら `done/` へ **git mv** で移動する。
3. 1タスクは1セッションで終わる粒度に分割する。
4. **セッション終了時に必ず `STATUS.md` を更新する。** これを怠ると次のセッション（別のAIかもしれない）が再開できない。
5. `STATUS.md` は**薄い索引**に保つ。作業の詳細（試したこと・ハマりどころ）は各タスクファイルの「作業ログ」に書く。タスクファイルは1タスク1ファイルなので、複数人・複数エージェントが並行してもコンフリクトしない。
