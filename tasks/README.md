# tasks/ — タスク管理

「**今なにをしているか**」を管理する場所。GitHubのIssueや外部タスクツールに依存せず、リポジトリ内で完結させる（SSOT原則）。

## 構成

```
tasks/
├── STATUS.md      # 現在地スナップショット（毎セッション更新・最重要）
├── TEMPLATE.md    # タスクファイルの雛形
├── backlog/       # 未着手のタスク
├── in-progress/   # 進行中のタスク（原則1つまで）
└── done/          # 完了したタスク
```

## 運用ルール

1. タスクは `TEMPLATE.md` をコピーして `backlog/` に作る。ファイル名: `YYYYMMDD-短い英語スラッグ.md`（例: `20260706-todo-app-mvp.md`）。
2. 着手したら `in-progress/` へ、完了したら `done/` へ **git mv** で移動する。
3. 1タスクは1セッションで終わる粒度に分割する。
4. **セッション終了時に必ず `STATUS.md` を更新する。** これを怠ると次のセッション（別のAIかもしれない）が再開できない。
5. 作業中の気づき・ハマりどころ・試して失敗したことは、タスクファイルの「作業ログ」に書く。会話履歴に置き去りにしない。
