# 仕様: Navigator v1（Context Budget運用の最小セット）

- **作成日**: 2026-07-13
- **最終更新**: 2026-07-13
- **ステータス**: 実装済み（2026-07-13 再現テスト合格・正式採用）
- **関連**: `AGENTS.md` §2・§4、`templates/task.md`、`prompts/review-code.md`、`docs/philosophy/design-principles.md`

## 1. 目的（なぜ作るのか）

実装AIのセッションが、要件定義・壁打ち・レビュー往復・探索的な全体読み込みでコンテキストを浪費する問題を解決する。Navigator v1は新しいアプリではなく、**人とAIの分業ルール（交通整理）の正本**であり、実体は既存3ファイルへの追記である。導入後は「構想は実装の外で・実装は限定された入力で・検品は差分だけで」が誰の運用でも再現される。

## 2. スコープ

### やること（3つだけ）

- **Manifest**: タスクファイルに「AIが読むもの」を明示する欄とプリセット3種を設ける
- **Reviewer**: 差分と仕様該当節だけを入力に、合否・指摘・次に貼る修正プロンプトを返すレビュー用プロンプトを整備する
- **再開ブロック**: セッション終了時に「次に貼るもの＋貼り先」まで書く引き継ぎ書式を定める

### やらないこと（明示的に対象外）

- 新フォルダ・新アプリ・新ロールの追加（Architect / Project Manager / UX Advisor は作らない）
- HTML・UI・Portal連携・自動化スクリプト
- 特定AIベンダー向けの最適化・分岐（本文は「実装AI／レビューAI／構想AI」の3語のみで書く）
- 消費率計測の自動化

## 3. ユーザーから見た振る舞い（運用の1周）

```
1. 構想AI（実装の外のチャット）で壁打ちし、結論を仕様に固める（人間が運ぶ）
2. タスク起票: プリセットを選んで Context Manifest を1〜2行差し替える
3. 実装AIセッション: Manifest記載のファイルだけを読んで実装・テスト・コミット
4. 人間が diff をコピーし、レビューAI（prompts/review-code.md を貼った窓）へ貼る
5. 返ってきた修正プロンプトを実装AIへそのまま貼る（合格ならマージへ）
6. どのセッションも、終了時に STATUS の再開ブロックを更新して閉じる
```

どの工程で寝かせても腐らない（正はチャットではなくリポジトリにある）。1日1回しか触らない人でも、再開ブロックの5行から記憶ゼロで再開できる。

### 4-C. `prompts/review-code.md` の改修（全文を以下へ置き換え）

```markdown
# プロンプト: 差分レビュー（レビューAI用）

> 実装AIとは**別の窓（チャット）**に、このファイル全文を貼る。レビューAIがリポジトリを読める必要はない — 差分と仕様の該当節を貼るだけでよい。

---

あなたはこのプロジェクトの**レビューAI**です。受け取るのは「差分（diff）」と「仕様の該当節」だけ。返すのは「合否・指摘・次に実装AIへ貼る修正プロンプト」の3点です。あなたは実装をしません。

まず「レビューAIとして待機します。diffと仕様の該当節を貼ってください」と名乗り、入力を待ってください。

## 入力（利用者が貼るもの）

1. レビュー対象の差分（`git diff` の出力、またはPRの差分）
2. 対応する仕様書の該当節（全文は不要）

## 診断の観点

- **仕様との一致**: 「やること」を満たすか。「やらないこと」に踏み込んでいないか。「完成の定義」を満たすか
- **正しさ**: バグ・エッジケースの見落とし（空入力・境界値・異常系）はないか
- **セキュリティ**: 秘密情報の混入、入力の未検証、XSSなどの基本的な脆弱性はないか
- **保守性**: 過剰な複雑さ、不要な依存、差分から読み取れる範囲のスタイル不整合はないか

差分と仕様から確認できないことは「未確認」と明記する。憶測での指摘、確認していない観点を確認済みと報告することは禁止。

## 出力（必ずこの3部構成・この順番）

1. **合否**: 合格（このまま進めてよい）／不合格（要修正）
2. **指摘リスト**: 1件ずつ「温度（must / should / nit）・場所（ファイル名と行）・問題（どんな入力・操作で何が壊れるか）・提案（どう直すか）」
3. **修正プロンプト**: must と should を反映した、実装AIにそのまま貼れる指示文を**1つのコードブロック**にまとめる（利用者が全選択→コピーするだけで済む形）。合格の場合は「修正プロンプトなし」と書く
```
