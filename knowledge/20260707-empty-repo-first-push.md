# 空のリポジトリへの最初のpushで知っておくこと2つ

- **状況**: 作りたてで空（コミットゼロ）のGitHubリポジトリに、はじめてpushした
- **結論**: ①最初にpushしたブランチが自動で**デフォルトブランチ**になる ②`git ls-remote` が何も返さなくてもエラーではない（空だから正常）
- **理由**: GitHubは最初のref（ブランチ）をデフォルトに設定する仕様。空リポジトリには参照するものが無いので一覧も空になる

## 詳細

- 作業ブランチから始めた場合、あとで `main` を作ってデフォルトを切り替える一手間が必要になる（`20260707-change-default-branch.md`）。**最初から `main` にpushして、そこから作業ブランチを切るほうが楽**
- 空リポジトリではPRも作れない（`20260707-pr-needs-two-branches.md`）
- GitHub APIの一部（ツリー操作系）も空リポジトリでは失敗することがある。最初の1コミットはgit pushで入れるのが確実

## 実際に起きた日

2026-07-06〜07。このリポジトリ自身の立ち上げ時。
