## Context Manifest — このタスクでAIが読むもの

下のプリセットから1つ選んで残し、`<アプリ名>` 等を差し替える。**ここに無いファイルをAIは読まない。**

**プリセットA: 新規アプリ実装**

| 区分 | ファイル | 範囲 |
|---|---|---|
| 必読 | tasks/STATUS.md | 再開ブロックのみ |
| 必読 | このタスクファイル | 全文 |
| 必読 | apps/<アプリ名>/specs/<仕様書>.md | 全文 |
| 参照可（必要時のみ） | docs/coding-standards.md | 実装言語の節 |
| 参照可（必要時のみ） | templates/app/ | 雛形として |

**プリセットB: 既存アプリ修正** — 必読: STATUS再開ブロック／このタスクファイル／対象仕様書の該当節／変更対象ソースのみ。参照可: docs/coding-standards.md の該当節

**プリセットC: ドキュメント作業** — 必読: STATUS再開ブロック／このタスクファイル／対象ドキュメント。参照可: templates/ の該当テンプレート

**読まない（全プリセット共通）**: docs/philosophy/・docs/reviews/・knowledge/・他アプリ・README群。思想判断が必要になったら、読まずに質問として持ち帰り停止する。

**調整欄**（使うツールの計測単位に合わせて書き換える。計測できないツールでは変更ファイル数とタスク分割で代替）:
- 開始時の読み込み目安 10%以内 / 作業目標 50% / 警戒線 70%
- セッション終了時、実測と主因を作業ログへ1行記録する

## 再開ブロック書式（セッション終了時に tasks/STATUS.md の「進行中のタスク」へ書く）

    ### 再開ブロック（YYYY-MM-DD）
    - 現在地: （どのタスクのどこまで終わったか）

    - 読むもの: （このブロック＋タスクファイル＋Manifestの必読のみ）
    - 触るな: （完成済み・再解釈禁止のもの）
    - 前回消費: （実測と主因を1行）
