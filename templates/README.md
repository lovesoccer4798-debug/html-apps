# templates/ — テンプレート置き場

ワークスペースで使うすべてのテンプレートをここに集約する。

| テンプレート | 用途 | コピー先 |
|---|---|---|
| `spec.md` | 仕様書 | `apps/<アプリ名>/specs/<機能名>.md` |
| `task.md` | タスク | `tasks/backlog/YYYYMMDD-<アプリ名>-<スラッグ>.md` |
| `adr.md` | 設計判断の記録 | `docs/adr/YYYYMMDD-<スラッグ>.md` |
| `app/` | 新規アプリの雛形一式 | `apps/<アプリ名>/`（`cp -r templates/app apps/<アプリ名>`） |

## 仕様書の運用ルール

- 仕様は**生きたドキュメント**。実装中に判明した変更は仕様書に反映し、実装と仕様を乖離させない。
- 「どう実装するか」は書かない（それはコードとADRの領分）。「何を・なぜ・完成の定義」を書く。
- 廃止した仕様は削除せず、冒頭に「ステータス: 廃止」と理由を書いて残す。

## テンプレート自体の変更

テンプレートの変更は既存ドキュメントには遡及しない（書式が混在してよい）。大きな構造変更をする場合はADRを残すこと。
